### Adobe Analytics 中的新功能 {#aa-features}

| 功能 | 說明 |
| -----------| ---------- |  
| 地區資料收集的變更 | 為改善資料收集效能，所有其餘客戶都已轉換為地區資料收集(RDC)，如此收集作業就會盡可能接近您的使用者。 這可改善網站／應用程式效能，並確保資料能盡快收集，以最佳化使用者體驗。<br/>按地區收集您數位財產中的資料後，系統會透過安全連線將資料轉送到資料處理中心，資料經過處理後，便可供 Adobe Experience Cloud 的產品使用。十幾年來，新的實作項目一律預設採用 RDC。[更多內容](https://marketing.adobe.com/resources/help/zh_TW/reference/regional-data-collection.html) |
| 中國的 RDC 異動 | Adobe在中國大陸的地區資料收集功能可讓中國境內客戶直接將資料傳送至中國境內的資料收集中心，而非全球其他位置。 相較於將資料傳送至中國境外的資料收集中心，這可改善頁面載入時間和資料準確度。 [更多內容](https://marketing.adobe.com/resources/help/zh_TW/reference/rdc-china.html) |
| 中國區域加速 | 「Analytics報表套裝設定」現在提供「中國加速」選項。 這個選項可改善中國大陸使用者的Adobe Analytics Reporting使用體驗。 每個Adobe Analytics使用者都可以啟用或停用此選項。 請勿針對中國大陸以外的使用者選取此選項，因為這會對效能產生負面影響。 [更多內容](https://marketing.adobe.com/resources/help/zh_TW/sc/user/report_settings.html) |
| Livestream 新變數 | * **carrier**: 行動裝置的無線服務提供者。<br/>* **tnt:** Target 中使用的測試與目標欄位。_actionId_ 與 _actionName_ 從未有值。<br/>* **tntAction:** Target 中使用的加強測試與目標欄位。與 tnt 欄位不同之處在於，_actionId_ 和 _actionName_ 會設定值。 |
| API 2.0: JWT 驗證支援 | Adobe Analytics 2.0 API 現起支援 JWT 驗證。如需 JWT 驗證的詳細資訊，請前往: [https://github.com/AdobeDocs/analytics-2.0-apis/blob/master/jwt.md](https://github.com/AdobeDocs/analytics-2.0-apis/blob/master/jwt.md) |

### Media Analytics SDK 2.2，適用於: Chromecast、Roku、Apple TV 和 [!DNL Launch]Web Extension {#aa-va}

| 功能 | 說明 |
| -----------| ---------- |
| 適用於音訊的 Adobe Analytics | 將聽眾的串流音訊參與度和完整的數位行為連結在一起。您可以在通常較為分散的音訊市場中，掌握聽眾身分以及收聽地點、時間和頻率，並評估使用者行為。<br/>此功能可讓企業衡量觸及的聽眾範圍和內容受歡迎的程度，並根據使用者行為和類似受眾模型，提供讓使用者持續參與的深入分析。使用我們同級最佳的SDK評估10秒音訊內容和1秒串流廣告，以評估體驗品質（停頓、錯誤、開始時間）和內容加上廣告KPI。 |
| 維護廣告插播 | 在播放器已輸入廣告插播，且特定時段內未傳送呼叫的情況下，增強廣告追蹤，SDK邏輯會預設為內容播放，即使播放器仍處於廣告插播狀態亦然。 在前段廣告期間，這可能會在廣告中斷完成前引發內容開始引發。 在Media 2.2 SDK中，當廣告之間有長時間延遲時，會在廣告中斷內的廣告之間傳送緩衝呼叫。 |
| SDK 名稱變更 | 從2.2版開始，視訊心率程式庫(VHL)SDK會重新命名為Media SDK。 Media 2.2 SDK完全向後相容於VHL 2.X SDK系列。 名稱變更不代表功能中斷，只代表命名慣例的變更。 |
