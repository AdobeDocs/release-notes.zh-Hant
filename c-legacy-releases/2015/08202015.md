### 報告及分析修正 {#fixes_reports_analytics}

* 使用「選取報表套裝和變數」面板 [更新分類規則產生器](https://docs.adobe.com/content/help/en/analytics/components/classifications/classifications-rulebuilder/classification-rule-definitions.html) 。 此面板可讓您新增報表套裝和變數至現有的規則集。 以前，您在建立規則集時已指定變數和報表套裝。 此變更可防止分類變數發生報告的問題。
* 已修正建立和編輯區段時發生的權限問題。 (AN-106175)
* 修正當使用者在登入頁面上勾選「記住我」後，登入並嘗試修改透過連結傳送之報表時所發生的問題。 這會導致「存取遭拒」錯誤。 (AN-89504 + AN-107008)
* 修正計算量度報表上每小時警報無法正常運作的問題。 (AN-105946)
* 修正「檢視所有報表」搜尋功能在IE 11中無法運作的問題。 (AN-104672 + AN-102566)
* 修正處理規則中無法使用特殊字元（UTF-8和多位元組）的問題。 (AN-102095)
* 修正「產品」報表中遺失「產品檢視」度量的問題。 (AN-106692)
* 修正傳送頻率為「每年」的報表無法排程的問題。 (AN-105562)
* 修正無法傳送含多位元組字元的排程控制面板的問題。 (AN-106309)
* 修正區段和使用者管理介面無法載入的問題。 (AN-106627)
* 修正無法顯示「區段詳細資料」的問題。 (AN-105185)
* 修正每小時警報觸發錯誤的問題。 (AN-97573 + AN-105946)
* 修正建立包含計算量度的目標時，會導致錯誤的問題。 (AN-104410)
* 修正趨勢報表中計算量度的問題。 (AN-105606)
* 修正每日警報與「關鍵量度」報表中顯示的值不符的問題。 (AN-105953)

### Ad Hoc Analysis 修正 {#fixes_ad_hoc}

* 修正的問題：使用者無法在 Ad Hoc Analysis 的「區段管理員」中與其他特定的使用者共用區段。(AN-105679)
* 修正計算量度的名稱（非定義）變更後發生，並導致傳回不同值的問題。 (AN-106214)
* 修正當使用「時間」作為量度類型時，臨機報表無法正確載入的問題。 (AN-105806)
* 修正階層檢視無法報告資料的問題。 (AN-103234)
* 修正的問題：Ad Hoc Analysis 的「區段產生器」沒有提供「分鐘」作為「依序分段」中過去某段時間內的選項。(AN-105632)
* 修正的問題：使用報表維度時，Ad Hoc Analysis 中的統計資料問題。(AN-105864)

### Data Warehouse 修正 {#fixes_data_warehouse}

* 針對登入頁面的量度計算，修正 Reports &amp; Analytics 和 Data Warehouse 之間「交易資料來源」資料的不一致。(AN-102263)
* 修正某些點擊無法獲得正確參與率評分的問題。 (AN-106759)
* 修正的問題：將包含「首次/上次接觸渠道」報表分類的區段套用至 Data Warehouse 報表時，無法擷取任何資料。(AN-89294)
* 修正當欄值大於2KB時造成當機的問題。 (AN-106201、AN-106850)
* 修正一個 Tableau 檔案問題，該問題導致檔名中包含 [!DNL  .csv] 時無法新增 *`.csv`* 副檔名。(AN-106466)
* 修正Unicode欄位中Tableau檔案的UTF-8字串處理問題。 (AN-106467)

### 點按資料流饋送修正 {#clickstream}

* **更新通知:**&#x200B;我們正在更新資料饋送的傳送基礎結構。我們正在新增額外的IP位址至已用於傳送資料饋送的儲存區。 如果您的FTP網站上有資料饋送傳送至的IP白名單，您需要將這些額外的IP位址新增至該白名單。
   * 倫敦：185.34.189.0/27，計畫於2015年8月13日新增
   * 奧克蘭：192.243.248.32/27，計畫於2015年8月20日新增
   * 達拉斯：66.235.153.96/27，計畫於2015年8月27日新增

* 如果使用 Amazon S3 傳送資料，新實施的 [!UICONTROL BucketOwnerFullControl] 功能會指定跨帳戶上傳的權限，因此使用者能在貯體中建立屬於完全不同帳戶的物件。Amazon Web Services (AWS) 帳戶擁有者建立貯體，並建立擁有在貯體中建立物件權限的使用者，接下來提供其他使用者建立物件的認證。如需 [此功能的完整資訊](https://marketing.adobe.com/resources/help/zh_TW/sc/clickstream/datafeeds_configure.html) ，請參閱設定資料饋送。

**修正、改良及淘汰內容**

* 修正Firefox的錯誤，此錯誤會讓您無法設定將區段傳送至目的地的開始日期。 (AAM-15785)

* 修正[!UICONTROL 特徵建立工具]的錯誤，在[!UICONTROL 運算式產生器]和[!UICONTROL 程式碼檢視]之間切換時，此錯誤會讓程式碼檢視無法正確顯示特徵元件。(AAM-15919、AAM-20121)
* 修正[!UICONTROL 訪客交易市場]的錯誤，即使沒有啟用模型化，此錯誤也會將資料饋送模型化。(AAM-16495)
* 在目標的基本資訊中，[!UICONTROL 資料匯出]標籤會針對套用至目標的匯出控制項，顯示好記的名稱。(AAM-19824)
* 修正多個[!UICONTROL 區段產生器]錯誤，這些錯誤會使產生器無法：
   * 在編輯區段後傳回更新的區段大小資料。 (AAM-15809)

   * 傳回30天間隔的特徵大小。 (AAM-15982)

   * 在建立複雜規則時重新計算估計的區段大小。 (AAM-20265)

* 修正多個[!UICONTROL 訪客交易市場]錯誤：
   * 資料購買者不會再看到測試或示範訂閱者帳戶。 (AAM-11819)

   * 訂閱饋送的文氏圖表會以頻率計數（而非%）顯示區段與供應商區段之間的重疊。 (AAM-20495)
